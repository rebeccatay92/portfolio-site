Marco

Travel content page
Dashboard with list of itineraries
Adding / editing events
Itinerary map view
User's media console
Marco is a prototype itinerary planner and travel content site. Users are able to plan itineraries and view information about trips others are making. The front-end is built in React with Redux for state management and Apollo for data-fetching.
I converted wireframes into React components and implemented desired user flows. For the map interface of our planner, I integrated maps, location search and geocoding APIs with custom filters for markers, and a sidebar to view planned activities.
I immplemented user authentication and authorization using an OAuth2.0 API and dynamic routing, and experimented with user uploads of media content to Google Cloud Storage via a media control component.
On the backend, I set up a GraphQL-Express-Postgres server which allows for flexible querying of the database. Using custom types and type resolution allowed our backend to assemble and output data in the structure frontend needed for rendering. I also refactored our GraphQL resolvers from promises to async/await, resulting in less code and improved readability.

KryptoKnight is a cryptocurrency educational tool. There is a real time chart with basic technical indicators, and a virtual brokerage for users to gain exposure to cryptocurrency trading. In this group project, I focused on external APIs and fetched price data from Cryptocompare, calculated the technical indicators and charted them with MetricsGraphics.js

Magic Mango

Landing page
Custom hero build
Browse by heroes
MagicMango is an emulation of popular DOTA2 strategy sites, where users can create and share hero item builds depending on playstyle and the current metagame. I fetched data from the DOTA2 API and built an item selection page where users can click on items to add or remove them from their current build list; similar to how the in-game item guide works. Users can also browse by hero and see what others have created.
